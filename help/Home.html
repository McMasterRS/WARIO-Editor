Flowchart Interface

Shortcuts

  • Ctrl+S - Save*
  • Ctrl+R - Save & Run (WIP)*
  • Ctrl+L - Load*
  • Ctrl+G - Global Variables*

Manipulating Nodes

  • Right click - Add new node
  • Ctrl+D - Duplicate selected node(s)*
  • Double click on node - Modify node settings
  • S - Snap selected node to grid

Navigating the Flowchart

  • F - Focus on selected node(s)
  • Alt+Left click - Drag flowchart
  • Scroll wheel - zoom in/out

All commands marked with a * can also be found in the menu bar

Creating New Nodes

To create a new node, right click on the main interface, click the toolbox you wish to create a node from, click the category of node you want to create and then select the node itself.

Connecting Nodes

Clicking on the circle next to an attribute name and dragging allows you to connect it to other nodes' attributes.

Attributes will automatically grey out if they are incompatible with the one you're attempting to link from. Each node's attributes are assigned data types and only attributes with matching types can be connected.

Additionally, you can hover over the attribute connectors to see the attribute type.

Adding Custom Nodes

To add a custom node, right click on the flowchart interface and select "Custom Node". This will create an empty node with no attributes.

Double clicking on this new node will open the settings window where the custom node file can be selected. This will immediately populate both the node's attributes and settings. Loading a new file will overwrite any settings entered. These settings will be preserved like any other node upon saving and loading.

Adding Global Variables

Global variables can be accessed by all nodes during pipeline execution. They are defined in the global variables window, accessible in the Edit menu or by the shortcut given above.

The buttons at the bottom can be used to add or remove global variables. Be aware that removing a global variable will remove any connections from nodes using that variable. Each variable has 4 components:

  • A Unique name
  • A Data type selector
  • The Data value
  • A checkbox to make the variable a constant

The data type of the variable sets the type of the attribute when used in a get or set node (see below). A standard set of varibles is given but by selecting "custom" any type can be used. If the "file" type is selected, the value must contain the file location. Global variables with the "File" type can be connected to nodes of any data type.

Attempting to use the same name for two variables will revert the modified variable to its original name. Modifying a global variable's name or type will remove all connections so as not to create invalid paths.

Toolkit Global Variables

Some toolkits include default global variables that are automatically added when the toolkit is enabled. These global variables can only be removed when the toolkit is disabled. Their name, type and constant bool are fixed but the value held by them can be changed at will. These variables appear as greyed out, as seen below.

Including Global Variables In Flowcharts

You can use global variables in your flowcharts by creating "Get Global" and "Set Global" nodes which retrieve and modify global variables respectively. These are found in the "input" and "output" categories of the default toolbox and function in the same way as any other node, with a setting that allows you to select the global variable for the node.

Global variables are passed to all nodes when ran so custom nodes can make use of them without requiring the get/set nodes. See Custom Nodes for more information.

BE AWARE: There is no guarantee what order the flowchart will be computed other than no node running until all of its input nodes have ran. This means that using the "Set Global" node to make a change to a global variable that another node relies on could result in undefined behavior.
